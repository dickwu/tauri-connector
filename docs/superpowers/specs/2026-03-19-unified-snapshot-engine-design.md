# Unified Snapshot Engine — Design Spec

## Problem

The current DOM tree capture in tauri-connector has significant gaps that degrade Claude's ability to understand and interact with Tauri apps, especially those built with React + Ant Design:

1. **Incomplete trees**: No shadow DOM traversal, no iframe handling, no portal stitching. Ant Design modals/drawers/dropdowns render at `document.body` via React portals, appearing disconnected from their trigger elements.
2. **Performance**: Recursive `el.children` traversal risks stack overflow on deep React trees and is ~78x slower than the native TreeWalker API.
3. **AI readability**: No React component name enrichment (CSS-in-JS hash classes are meaningless), no virtual scroll detection (Claude thinks lists are empty), no visibility pruning (hidden elements clutter the tree).
4. **Reliability**: No graceful truncation for large DOMs, no retry with backoff for dynamic content.
5. **Code duplication**: Three separate walker implementations (`buildAccessibilityTree` + `buildStructureTree` in bridge.rs, IIFE in snapshot.rs) with inconsistent features.

## Approach

**Unified Snapshot Engine** — a single JS snapshot function replacing all three walkers, inspired by Playwright's `ariaSnapshot` architecture but adapted for Tauri + React/Ant Design. Key research findings informing this design:

- Playwright's modern `ariaSnapshot` is 100% injected JavaScript (not CDP), confirming JS injection is the correct approach
- Tauri/WKWebView offers no native DOM access beyond JavaScript execution — macOS AX API is blocked by sandboxing
- D2Snap research shows hierarchy is critical for agent performance — flat element lists degrade accuracy even with lower token count
- TreeWalker API benchmarks at ~0.9ms vs ~70ms for recursive descent on 1000-element DOMs
- React fiber nodes (`__reactFiber$xxx`) are accessible on DOM elements after render, giving component names without timing requirements

## Architecture

### Unified Function

A single JS function `window.__CONNECTOR_SNAPSHOT__(options)` replaces all existing walkers. Generated by `build_snapshot_engine()` in `plugin/src/bridge.rs`.

```javascript
window.__CONNECTOR_SNAPSHOT__(options)
// options = {
//   mode: "structure" | "accessibility" | "ai",
//   selector: string | null,
//   maxDepth: number,             // 0 = unlimited
//   maxElements: number,          // 0 = unlimited
//   reactEnrich: boolean,
//   followPortals: boolean,
//   shadowDom: boolean,
// }
//
// Returns: {
//   snapshot: string,
//   refs: Record<string, RefEntry>,
//   meta: { elementCount, truncated, portalCount, virtualScrollContainers }
// }
```

### Mode Behavior

| Mode | Includes | Ref IDs | Use case |
|---|---|---|---|
| `"structure"` | tag, id, classes, data-testid | No | Debugging layout |
| `"accessibility"` | role, name, ARIA states | No | Understanding semantics |
| `"ai"` | role, name, states, component name, refs, cursor=pointer | Yes | Claude interaction (default) |

### Output Format

YAML-like indented text, same spirit as Playwright's aria snapshots:

```yaml
- dialog "Edit Appointment" [ref=e12, component=AppointmentModal]:
  - heading "Edit Appointment" [level=2]
  - textbox "Patient Name" [ref=e13, required]
  - combobox "Status" [ref=e14, component=InternalSelect, expanded=true]:
    - listbox "Status options" [portal]:
      - option "Active" [selected]
      - option "Inactive"
  - list [virtual-scroll, visible=8]:
    - listitem "Recent visit" [ref=e20]
```

## DOM Walker

### TreeWalker Replacement

Replace recursive `el.children` with `document.createTreeWalker`:

```javascript
const walker = document.createTreeWalker(root, NodeFilter.SHOW_ELEMENT, {
  acceptNode(node) {
    // REJECT prunes entire subtree (no children visited)
    if (['SCRIPT','STYLE','NOSCRIPT','TEMPLATE'].includes(node.tagName))
      return NodeFilter.FILTER_REJECT;
    if (node.getAttribute('aria-hidden') === 'true')
      return NodeFilter.FILTER_REJECT;

    const style = getComputedStyle(node);
    if (style.display === 'none')
      return NodeFilter.FILTER_REJECT;
    if (style.visibility === 'hidden')
      return NodeFilter.FILTER_REJECT;

    // SKIP collapses this node (children still visited)
    const role = node.getAttribute('role');
    if (role === 'presentation' || role === 'none')
      return NodeFilter.FILTER_SKIP;

    return NodeFilter.FILTER_ACCEPT;
  }
});
```

Depth tracking via parent stack (TreeWalker is flat).

### Visibility Pruning

| Condition | Action |
|---|---|
| `aria-hidden="true"` | REJECT (prune subtree) |
| `display: none` | REJECT (prune subtree) |
| `visibility: hidden` | REJECT (may over-prune rare visible-child cases; acceptable trade-off for O(n) performance) |
| `role="presentation"` or `role="none"` | SKIP (collapse, visit children) |
| `SCRIPT`, `STYLE`, `NOSCRIPT`, `TEMPLATE` | REJECT |

### Portal Stitching (Two-Pass)

The snapshot builds an intermediate tree structure (not direct string output) during Pass 1, then stringifies in a final render phase. This allows portal subtrees to be inserted at arbitrary logical positions.

**Pass 1 — Main walk**: Traverse the DOM with TreeWalker, building an in-memory tree of `{ role, name, attrs, children, depth }` nodes. At each node with `aria-controls` or `aria-owns`, record `{ treeNode, targetId }` in a `portalLinks` array. Mark any `document.body` direct children that match a recorded `targetId` as "claimed" (skip them in the main tree to avoid duplication).

**Pass 2 — Portal walk**: For each recorded portal link, create a **second TreeWalker** rooted at the target container element (`document.getElementById(targetId)`). Walk this subtree with depth rebased to `trigger_depth + 1` (the trigger node's depth + 1 becomes depth 0 of the portal subtree). Attach the resulting subtree as children of the trigger's tree node, with a `[portal]` annotation.

**Depth rebasing rule**: Portal children inherit the trigger's indentation level. If a trigger combobox is at depth 4, its portal listbox renders at depth 5, and options at depth 6 — regardless of the portal DOM element being a direct child of `document.body` (depth 1 in the real DOM).

**Render phase**: After both passes, the in-memory tree is stringified to the indented YAML-like text format in a single recursive pass over the tree structure.

**Orphan portal fallback**: Direct children of `document.body` that were NOT claimed by any `aria-controls`/`aria-owns` link but match Ant Design heuristics (have `ant-` prefixed classes, or `rc-` prefixed classes like `rc-dialog-root`) are annotated as `[orphan-portal]` and appended at the end of the output. This covers Ant Design components that lack ARIA linkage between trigger and portal (e.g., older Modal/Drawer versions, custom `getPopupContainer` targets).

### Shadow DOM Traversal (Opt-In)

When `shadowDom: true`, after visiting an element's children, also walk `el.shadowRoot.children`. Not needed for Ant Design (no shadow DOM), but supports web component apps.

### Virtual Scroll Detection

Elements with class `rc-virtual-list-holder` (from `@rc-component/virtual-list` used by Ant Design Select, Table, Tree) are annotated:

```yaml
- list [virtual-scroll, visible=N]:
```

Where N = number of visible child items in the holder-inner container.

## React Fiber Enrichment

When `reactEnrich: true`. The `fiberKey` variable lives in the **outer IIFE scope** that registers `window.__CONNECTOR_SNAPSHOT__`, NOT inside the snapshot function body. This ensures the `Object.keys()` scan runs only once per page load (the key suffix is stable for the lifetime of the page), and all subsequent snapshot invocations (including `autoPushDom`) reuse the cached key:

```javascript
// In the outer IIFE (runs once at bridge injection time):
let fiberKey;  // undefined = not yet looked up, null = not a React app

// These helpers are in outer scope, shared across all snapshot calls:
function findFiberKey(el) {
  if (fiberKey !== undefined) return fiberKey;
  fiberKey = Object.keys(el).find(k => k.startsWith('__reactFiber$')) || null;
  return fiberKey;
}

function getComponentName(el) {
  const key = findFiberKey(el);
  if (!key) return null;
  let fiber = el[key];
  while (fiber) {
    const t = fiber.type;
    if (t && typeof t === 'function') {
      return t.displayName || t.name || null;
    }
    fiber = fiber.return;
  }
  return null;
}

// Snapshot function closes over fiberKey:
window.__CONNECTOR_SNAPSHOT__ = function(options) {
  // ... uses getComponentName(el) which reads fiberKey from closure
};
```

- Fiber key suffix cached in outer IIFE scope (persists across all calls, scanned once per page load)
- Skip generic wrappers: `App`, `Layout`, `ConfigProvider`, `ThemeProvider`, `Fragment`, `Suspense`, `ErrorBoundary`, `StrictMode`
- Output as `component=Name` attribute on the node

## Accessible Name Computation

Improved toward W3C accname-1.2:

| Case | How resolved |
|---|---|
| `aria-labelledby` with multiple IDs | Space-joined text of all referenced elements |
| `aria-label` | Direct value |
| `<label for="id">` | Label text for inputs |
| `<img alt>` | Alt text |
| Button/link text | Visible `textContent` (not hidden) |
| `<input type="submit/reset">` | `value` attribute, then type default |
| `<fieldset>` + `<legend>` | Legend text |
| `<figure>` + `<figcaption>` | Figcaption text |
| `<table>` + `<caption>` | Caption text |
| `placeholder` | For inputs without label |
| `title` attribute | Last resort fallback |
| `::before`/`::after` content | `getComputedStyle(el, '::before').content` |

### ARIA State Coverage

| State | Detection |
|---|---|
| `checked` | `aria-checked` or `el.checked` |
| `disabled` | `aria-disabled` or `el.disabled` |
| `expanded` | `aria-expanded` |
| `selected` | `aria-selected` |
| `pressed` | `aria-pressed` |
| `level` | `aria-level` or heading tag number |
| `required` | `aria-required` or `el.required` |

### Updated DomEntry

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SnapshotMeta {
    pub element_count: usize,
    pub truncated: bool,
    pub portal_count: usize,
    pub virtual_scroll_containers: usize,
}

#[derive(Debug, Clone)]
pub struct DomEntry {
    pub window_id: String,
    pub html: String,
    pub text_content: String,
    pub snapshot: String,
    pub snapshot_mode: String,
    pub refs: RefMap,                // typed HashMap<String, RefEntry> — NOT raw JSON
    pub meta: SnapshotMeta,          // typed struct — NOT raw JSON
    pub timestamp: u64,
}
```

**Serialization boundary**: The JS engine returns `refs` as a JSON object. The Rust handler deserializes it to `RefMap` (`HashMap<String, RefEntry>`) immediately upon receiving the bridge response. This ensures `build_resolve_and_act_script(selector, &entry.refs, action_js)` continues to work without any call-site changes. The `RefEntry` struct, `RefMap` type alias, `parse_ref`, and `build_resolve_and_act_script` in `crates/cli/src/snapshot.rs` are unchanged.

**For Tauri IPC (auto-push path)**: The frontend's `autoPushDom()` sends `refs` as a JSON string. The `push_dom` command handler in `plugin/src/lib.rs` deserializes it to `RefMap` before storing in `PluginState.dom_cache`.

**For MCP responses**: `get_cached_dom` serializes `refs` and `meta` to JSON in the response body. `dom_snapshot` returns the snapshot string directly (refs are only cached, not included in the live snapshot response unless the caller requests them).

## Backward Compatibility

- `webview_dom_snapshot` `type` parameter maps to `mode` for backward compat: `"accessibility"` and `"structure"` work as before
- **`get_cached_dom` breaking change**: Response body changes from `{ accessibility_tree, structure_tree, ... }` to `{ snapshot, snapshot_mode, refs, meta, ... }`. This is a **semver minor bump** (0.4 -> 0.5). Old consumers reading `accessibility_tree` will get `undefined`. The new `snapshot` field contains the equivalent content in the new format.
- CLI `--interactive` maps to `mode: "ai"` (both produce ref-annotated output). `--compact` becomes a post-filter that keeps only lines containing `ref=` or a quoted accessible name.
- **Timeout**: The `dom_snapshot` handler timeout stays at 15 seconds. For `autoPushDom` (background, non-blocking), no explicit timeout since it uses Tauri IPC `invoke` which is fire-and-forget from the JS side.

## Token Budget and Truncation

When `maxElements > 0`, the walker stops emitting new lines after reaching the limit. The output ends with:

```yaml
# ... truncated (5000 of 12847 elements shown)
```

`meta.truncated` is set to `true` in the return value. This gives Claude a clear signal that the tree is incomplete.

## Not In Scope

- Incremental diff mode (MutationObserver-based delta transmit) — future enhancement
- Set-of-Mark screenshot overlay — future enhancement
- `__REACT_DEVTOOLS_GLOBAL_HOOK__` pre-load injection — requires timing that the bridge script can't guarantee
- WKWebView content world isolation — separate security enhancement
- iframe recursion — requires server-side orchestration not available in single JS execution
